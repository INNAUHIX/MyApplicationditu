# 鸿蒙足迹记录 App — 项目汇报（2026-07-09）

> 包名 `com.footprintapp.pro`，显示名仍为工程默认 "MyApplicationditu"，HarmonyOS 6.1.1，设备仅 phone。

## 一、在跑的东西（entry 模块，唯一启动入口）
- `EntryAbility` 加载 `pages/Index`（main_pages 仅注册此页）。
- `Index.ets`（约 1655 行）单文件承载全部功能：
  - 地图 MapKit（3D 建筑、昼夜模式）
  - 实时定位 `geoLocationManager` + WGS84→GCJ02 转换
  - RDB 持久化 `footprint.db` / `track_points(id,lat,lng)`，落库 + 重启恢复
  - 足迹算法：位移 ≥80m 记一个"足迹"点，全部点连轨迹；展示两种：折线轨迹 polyline / 海量点 MassPointOverlay（手搓红色圆点 PixelMap）
  - 权限流程：前台(模糊+精确) → 后台定位引导
  - 底部 Tab（HdsTabs）：足迹 / 我的；"我的"半模态内手写 设置页 + 隐私设置页（多数为 "功能不可用" 占位）
- `EntryAbility` 额外做后台保活（`onBackground` 起 location 后台任务）

## 二、已建但未接入工程的模块（关键发现）
根 `build-profile.json5` 只挂了 entry，根 `oh-package.json5` 无依赖。以下均不在编译链中：
- `feature/mine`：更规范的"我的/登录/关于/隐私"模块（@ComponentV2 + MineVM + route_map 路由）。**但残缺**：引用了不存在的 `MineVM`/`model/Index`/`constant/Index` 等；route_map 中 PrivacyPolicyPage/TermsOfServicePage/ProfilePage/SettingPage 的 .ets 未创建；文案还是模板的"综合工具模版"。无法编译，故未接入。
- `app_setting`：华为生态市场"通用应用内设置"库（深色模式/字体/清缓存/检测版本/关于/隐私/Web + AppServiceUtils）。功能齐全，但 Index.ets 没用它，自己手搓了一份设置 UI。
- `components/aggregated_login`：华为/微信/验证码聚合登录 + 隐私协议勾选 + 协议 WebView
- `components/collect_personal_info`：头像/日期选择
- `components/feedback`：意见反馈（http/api/mock/图片工具）
- `components/module_base`：HTTP 请求封装 + Mock 适配器 + UserService mock

## 三、问题与风险
1. **三套"我的/设置"实现并存**：Index 内嵌版、feature/mine 版、app_setting 库版，重复且不统一。
2. **feature/mine 是编译不过的半成品模板**，且未接入。
3. **MapPage.ets 是死代码**：未注册路由、无数据库、仅一个"开始记录"按钮，疑似早期原型。
4. **Index.ets 单文件过大**（~1655 行），地图/DB/权限/设置/隐私全挤一个 struct，难维护。
5. 大量设置项点击是 "功能不可用" 占位 toast。
6. 数据库存的是转换后的 GCJ02 坐标，原始 WGS84 未保留（如需原始 GPS 要调整）。
7. 应用显示名未品牌化，仍是 "MyApplicationditu"。

## 四、建议下一步
- 定架构方向：要么基于 feature/mine + app_setting 重构，要么在 entry 内收敛；删冗余。
- 补齐或移除 feature/mine（先移出工程避免迷惑，或补齐缺失文件改造成真模块）。
- 删除/接入 MapPage.ets。
- 拆分 Index.ets（地图页 / 数据层 / 设置页分离）。
- 明确后端边界（module_base 已有 mock）。
- 改应用名、补图标资源。

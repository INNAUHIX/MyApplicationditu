# 排查定位点不旋转问题计划

## 一、用户描述的问题

**用户反馈**:
- 原地转圈时,定位点(蓝点)**不像正常的定位点那样旋转**
- 正常的定位点应该**带有方向箭头,跟随手机朝向旋转**
- 用户怀疑:**"是不是添加了自己的定位代码没有使用官方的组件功能?"**

---

## 二、问题分析

### 根本原因: 自己的干扰代码

**当前代码**: [Index.ets 第137-165行](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L137-L165)

```typescript
private async enableMyLocation(): Promise<void> {
  // 1. 设置 style (FOLLOW_ROTATE)
  await this.mapController.setMyLocationStyle(style);
  
  // 2. 启用内置定位
  this.mapController.setMyLocationEnabled(true);
  
  // 3. 问题代码 - 我自己加的 cameraChange 监听
  this.mapEventManager?.on('cameraChange', (latLng: mapCommon.LatLng) => {
    this.recordTrackPoint(latLng);  // ← 干扰了相机控制
  });
}
```

**问题分析**:
- `cameraChange` 事件在相机每次变化时触发
- 在 `FOLLOW_ROTATE` 模式下,相机会**持续跟随手机方向旋转**
- 我的 `cameraChange` 监听记录了**每一次相机变化的位置**
- 这可能导致**相机被频繁重置,无法稳定显示旋转效果**
- 更重要的是:**这违反"使用官方方案"原则**——我加了自己的代码

### 核心原则

### 修复1: 移除 cameraChange 监听

**删除**: [Index.ets 第157-160行](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L157-L160)

### 修复2: 完全依赖 Map Kit 内置定位

修改 `enableMyLocation()` 方法:

```typescript
private async enableMyLocation(): Promise<void> {
  if (!this.mapController) {
    return;
  }

  try {
    // 1. 设置定位点样式 - 必须在 enabled 之前
    let style: mapCommon.MyLocationStyle = {
      anchorU: 0.5,
      anchorV: 0.5,
      radiusFillColor: 0x00000000,
      displayType: mapCommon.MyLocationDisplayType.FOLLOW_ROTATE
    };
    await this.mapController.setMyLocationStyle(style);

    // 2. 启用我的位置图层(显示带方向的蓝点)
    this.mapController.setMyLocationEnabled(true);
    this.mapController.setMyLocationControlsEnabled(true);

    // 3. 启动轨迹记录 - 用 Location Kit,不干扰 Map Kit
    this.startAutoTrack();
  } catch (err) {
    console.error(`enableMyLocation failed: ${err}`);
  }
}
```

### 修复3: 轨迹记录用 Location Kit(不干扰 Map Kit)

**新增** `startAutoTrack()` 方法:

```typescript
private startAutoTrack(): void {
  try {
    let requestInfo: geoLocationManager.LocationRequest = {
      priority: geoLocationManager.LocationRequestPriority.ACCURACY,
      scenario: geoLocationManager.LocationRequestScenario.NAVIGATION,
      timeInterval: 2,
      distanceInterval: 5
    };

    geoLocationManager.on('locationChange', requestInfo, (location: geoLocationManager.Location) => {
      // 转换坐标 WGS84 → GCJ02
      map.convertCoordinate(
        mapCommon.CoordinateType.WGS84,
        mapCommon.CoordinateType.GCJ02,
        location
      ).then((latLng: mapCommon.LatLng) => {
        // 关键: 只记录轨迹,不调用任何相机方法
        this.recordTrackPoint(latLng);
      });
    });
  } catch (err) {
    console.error(`startAutoTrack failed: ${err}`);
  }
}
```

**关键点**:
- `geoLocationManager.on('locationChange')` **不调用 moveCamera**
- **不干扰 Map Kit 的内置定位和相机控制**
- 职责分离: Map Kit 负责显示,Location Kit 负责数据

## 四、为什么 Map Kit 的定位点会不旋转?

### 可能原因 1: 我自己加的 cameraChange 干扰(主因)

我的 `cameraChange` 监听在每次相机变化时都会记录轨迹点,这可能:
- 频繁调用 `recordTrackPoint`
- 调用 `setPoints` 更新 polyline
- 间接影响相机的平滑旋转

### 可能原因 2: deviceType 缺少 sensor 配置

**检查 module.json5**: 是否声明了传感器权限

如果设备没有电子罗盘或磁力计,FOLLOW_ROTATE 可能退化为普通蓝点。

**但用户说"原地转圈"** —— 说明手机有方向传感器,只是显示不对。

### 可能原因 3: radiusFillColor 设为全透明

`radiusFillColor: 0x00000000` 让精度圈不可见,但**这不影响方向箭头**。

## 六、决策说明

### 为什么不用 cameraChange 记录轨迹?

- 用户的核心诉求:**"完全使用官方方案"**
- `cameraChange` 是**相机变化事件,不是定位点变化事件**
- 用户已经怀疑我加了自己的代码
- 移除它符合"完全官方"原则

### 为什么 Location Kit 不会和 Map Kit 冲突?

- Map Kit 内部**也用 Location Kit** 获取位置
- 外部注册 `on('locationChange')` **只是额外注册一个监听**
- **关键**: 不调用 `moveCamera` 就不会冲突
- 两个监听独立工作,互不干扰

### 为什么不显示轨迹开始点(初始)?

- 启动 App 时,Map Kit 自动定位,显示蓝点
- Location Kit 第一次回调时,记录第一个轨迹点(就是当前位置)
- 不会有"中间位置"的虚假点

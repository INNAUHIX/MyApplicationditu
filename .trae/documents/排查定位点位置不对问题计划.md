# 排查定位点位置不对问题

## 一、问题分析

用户报告"定位点定位不对,围绕的中心点看起来不太对"。

经过对比官方文档和当前代码,发现以下问题:

---

## 二、根本原因

### 问题1: mapOptions 中有不支持的配置

**代码位置**: [Index.ets 第35-52行](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L35-L52)

**当前代码**:
```typescript
this.mapOptions = {
  position: { ... },
  myLocationControlsEnabled: true,   // ← 问题:官方示例没有这个
  scaleControlsEnabled: true,        // ← 问题:官方示例没有这个
  compassControlsEnabled: true,      // ← 问题:官方示例没有这个
  zoomControlsEnabled: true,         // ← 问题:官方示例没有这个
  ...
};
```

**官方示例**(https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/map-location) 中的 mapOptions:
```typescript
this.mapOptions = {
  position: {
    target: { latitude: 39.9, longitude: 116.4 },
    zoom: 10
  }
  // ← 没有其他配置
};
```

这些控件(比例尺、指南针、缩放等)应该通过 controller 方法动态设置,而不是在 mapOptions 中。

---

### 问题2: 相机初始化和定位点位置不一致

**当前代码**: 地图初始位置在 (39.9, 116.4) 北京,但用户实际位置可能不是北京。

当启用 `setMyLocationEnabled(true)` + `FOLLOW_ROTATE` 后:
- 理论上,相机会自动跟随定位点,定位点应在屏幕中心
- 但**实际行为**: 相机可能不会自动跳到定位点(取决于内部实现)
- 表现: 用户看到蓝点不在屏幕中心(在某个偏离位置),而地图中心是初始位置(北京)

---

### 问题3: cameraChange 监听干扰了相机

**代码位置**: [Index.ets 第156-160行](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L156-L160)

**当前代码**:
```typescript
this.mapEventManager?.on('cameraChange', (latLng: mapCommon.LatLng) => {
  this.recordTrackPoint(latLng);
});
```

**问题**:
- `cameraChange` 在相机每次变化时都会触发
- 地图初始化时(从北京跳到定位点),会触发 cameraChange
- 用户拖动地图时,也会触发 cameraChange
- 这导致记录的轨迹位置是相机位置,不一定是定位点位置

---

## 三、修复方案

### 修复1: 简化 mapOptions

只保留 `position` 和 `dayNightMode`,移除其他配置:

```typescript
private initMapOptions(): void {
  this.mapOptions = {
    position: {
      target: {
        latitude: 39.9,
        longitude: 116.4
      },
      zoom: 10
    },
    dayNightMode: mapCommon.DayNightMode.NIGHT
  };
}
```

控件通过 controller 动态设置:
- `setMyLocationControlsEnabled(true)` - 定位按钮
- `setScaleControlsEnabled(true)` - 比例尺
- `setCompassControlsEnabled(true)` - 指南针
- `setZoomControlsEnabled(true)` - 缩放按钮

这些**在 enableMyLocation() 中调用**。

---

### 修复2: 启用 FOLLOW_ROTATE 模式

**关键**: `setMyLocationStyle` 必须在 `setMyLocationEnabled` 之前调用(这是 Map Kit 的强制要求)。

```typescript
private async enableMyLocation(): Promise<void> {
  if (!this.mapController) {
    return;
  }

  // 1. 先设置 style(必须!)
  let style: mapCommon.MyLocationStyle = {
    anchorU: 0.5,
    anchorV: 0.5,
    radiusFillColor: 0x00000000,
    displayType: mapCommon.MyLocationDisplayType.FOLLOW_ROTATE
  };
  await this.mapController.setMyLocationStyle(style);

  // 2. 启用我的位置图层
  this.mapController.setMyLocationEnabled(true);

  // 3. 启用各种控件
  this.mapController.setMyLocationControlsEnabled(true);
  this.mapController.setScaleControlsEnabled(true);
  this.mapController.setCompassControlsEnabled(true);
  this.mapController.setZoomControlsEnabled(true);

  // 4. 启动轨迹记录(用 Location Kit,不干扰相机)
  this.startAutoTrack();
}
```

---

### 修复3: 用 Location Kit 记录轨迹(不调用相机方法)

完全移除 `cameraChange` 监听,改用 Location Kit 单独获取位置:

```typescript
private startAutoTrack(): void {
  try {
    let requestInfo: geoLocationManager.LocationRequest = {
      priority: geoLocationManager.LocationRequestPriority.ACCURACY,
      scenario: geoLocationManager.LocationRequestScenario.NAVIGATION,
      timeInterval: 2,
      distanceInterval: 5
    };

    geoLocationManager.on('locationChange', requestInfo, (location: geoLocationManager.Location) => {
      // 转换坐标 WGS84 → GCJ02
      map.convertCoordinate(
        mapCommon.CoordinateType.WGS84,
        mapCommon.CoordinateType.GCJ02,
        location
      ).then((latLng: mapCommon.LatLng) => {
        // 只记录位置,不调用任何相机方法
        this.recordTrackPoint(latLng);
      });
    });
  } catch (err) {
    console.error(`startAutoTrack failed: ${err}`);
  }
}
```

**关键**: `geoLocationManager.on('locationChange')` **不调用任何 `moveCamera` 方法**,所以不会和 Map Kit 的内置定位冲突。

## 六、决策说明

### 为什么不用 cameraChange 记录轨迹?

- `cameraChange` 是相机变化事件,不是定位点变化事件
- 在用户拖动地图、缩放地图时都会触发
- 记录的是相机位置,不一定是定位点位置

### 为什么 Location Kit 不和 Map Kit 冲突?

- `geoLocationManager.on('locationChange')` 只注册监听,不调用任何相机 API
- Map Kit 内部也是用 Location Kit 获取位置,但有自己的相机控制逻辑
- 两者职责分离: Map Kit 控制相机,Location Kit 提供数据

### 为什么不在 mapOptions 中设置控件?

- 官方文档示例中,mapOptions 只设置 position
- 控件应该通过 controller 动态启用
- 这样更灵活,可以根据运行时条件启用/禁用

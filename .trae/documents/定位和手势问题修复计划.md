# 定位和手势问题修复计划

## 一、问题诊断

### 问题1: 放大缩小、定位功能全部失效

**根本原因**: 布局覆盖问题

**代码位置**: [Index.ets](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L244-L267) 第244-267行

**问题代码**:
```typescript
Column() {
  Blank()  // ← 撑满整个屏幕
  Button('清除轨迹')
}
.width('100%')
.height('100%')  // ← Column高度100%,覆盖全屏
```

**分析**: Column组件设置了`width('100%')`和`height('100%')`,加上内部的`Blank()`组件,导致整个屏幕被一个透明层覆盖,所有地图手势事件(缩放、拖动、点击定位按钮)都被这个透明Column拦截。

**解决方案**: 修改Column布局,只覆盖底部按钮区域,不拦截地图手势。

### 问题2: 定位中心点不对

**根本原因**: 双定位源冲突

**代码位置**:
- [Index.ets](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L141-L167) 第141-167行 `enableMyLocation()`
- [Index.ets](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L169-L207) 第169-207行 `startAutoTrack()`

**问题代码**:
```typescript
// Map Kit内置定位
this.mapController.setMyLocationEnabled(true);  // ← 定位源1
this.mapController.setMyLocationStyle({ displayType: FOLLOW_ROTATE });

// Location Kit自定义定位
geoLocationManager.on('locationChange', requestInfo, (location) => {
  // ← 定位源2,与Map Kit冲突
});
```

**分析**: 同时启用两个定位源:
1. Map Kit的内置定位系统(通过`setMyLocationEnabled` + `FOLLOW_ROTATE`)
2. 自定义的Location Kit监听器(通过`geoLocationManager.on('locationChange')`)

这两个定位源会同时争夺相机控制权,导致定位点跳动、中心点不稳定。

**解决方案**:
- 完全依赖Map Kit内置定位系统,移除自定义Location Kit监听
- 使用Map Kit的相机位置变化事件来记录轨迹点
- 这样既避免了定位冲突,又能记录轨迹

### 修复2: 定位冲突

**修改文件**: `entry/src/main/ets/pages/Index.ets`

**修改位置**: `enableMyLocation()` 方法

**修改内容**:

1. **移除自定义定位监听** - 删除`startAutoTrack()`方法

2. **添加Map Kit相机变化监听** - 在`enableMyLocation()`中添加:
```typescript
private async enableMyLocation(): Promise<void> {
  if (!this.mapController) {
    console.warn(TAG, 'mapController is null');
    return;
  }

  try {
    let style: mapCommon.MyLocationStyle = {
      anchorU: 0.5,
      anchorV: 0.5,
      radiusFillColor: 0x00000000,
      displayType: mapCommon.MyLocationDisplayType.FOLLOW_ROTATE
    };
    await this.mapController.setMyLocationStyle(style);
    console.info(TAG, 'MyLocationStyle set successfully');

    this.mapController.setMyLocationEnabled(true);
    this.mapController.setMyLocationControlsEnabled(true);
    console.info(TAG, 'My location enabled');

    // 监听相机位置变化来记录轨迹
    this.mapEventManager?.on('cameraChange', (cameraPosition) => {
      this.recordTrackPoint(cameraPosition.target);
    });

    this.moveToMyLocation();
  } catch (err) {
    let error = err as BusinessError;
    console.error(`Failed to enable my location, code: ${error.code}, message: ${error.message}`);
  }
}

private recordTrackPoint(latLng: mapCommon.LatLng): void {
  this.trackPoints.push(latLng);

  if (!this.trackPolyline) {
    let polylineOption: mapCommon.MapPolylineOptions = {
      points: this.trackPoints,
      width: 10,
      color: 0xFFFF0000,
      geodesic: false
    };
    this.mapController?.addPolyline(polylineOption).then((polyline) => {
      if (polyline) {
        this.trackPolyline = polyline;
      }
    });
  } else {
    this.trackPolyline.setPoints(this.trackPoints);
  }
}
```

**改动说明**:
- 完全移除`startAutoTrack()`方法和`geoLocationManager.on('locationChange')`监听
- 使用Map Kit的`cameraChange`事件来记录轨迹点
- 当定位点移动时,相机会跟随,我们记录相机位置作为轨迹点
- 避免了双定位源冲突,同时保持轨迹记录功能

## 五、技术决策说明

### 为什么使用`cameraChange`而不是`locationChange`?

**原因**:
1. Map Kit的内置定位系统会自动控制相机位置(因为使用了`FOLLOW_ROTATE`模式)
2. 当定位点移动时,相机会跟随移动,`cameraChange`事件会被触发
3. 使用`cameraChange`可以间接获取位置更新,而不需要额外的定位源
4. 这样可以完全避免双定位源冲突

### 为什么不保留Location Kit监听?

**原因**:
1. 项目记忆显示: "Simultaneous use of Map Kit's built-in positioning and custom Location Kit listeners causes location update conflicts"
2. 用户反馈"定位感觉有些奇怪围绕的中心点看起来不太对"正是双定位源冲突的表现
3. Map Kit的定位已经足够精确,无需额外的Location Kit监听

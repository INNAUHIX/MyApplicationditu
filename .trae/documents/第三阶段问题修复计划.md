# 第三阶段问题修复计划

## 一、问题分析

用户反馈的两个问题：
1. **定位中心点不对** - 围绕的中心点看起来不太对
2. **放大缩小、定位功能全部失效** - 无法正常使用

---

## 二、根本原因

### 问题1：定位冲突

**代码位置**：第141-167行 `enableMyLocation()` 和第169-207行 `startAutoTrack()`

**原因**：同时开启了两个定位源：
- Map Kit 自己的定位（通过 `setMyLocationEnabled(true)` + `FOLLOW_ROTATE`）
- 我自己注册的 Location Kit 定位（通过 `geoLocationManager.on('locationChange')`）

这两个定位源同时运行，会抢着更新位置，导致相机跳来跳去。

**解决方案**：
- 移除 `startAutoTrack()` 中自己注册的 `geoLocationManager.on('locationChange')`
- 使用 Map Kit 的 `on('myLocationButtonClick')` 或 `on('locationChange')`（Map Kit 自带的）来获取位置更新
- 或者：只用于记录轨迹，不干扰相机移动

---

### 问题2：手势失效

**代码位置**：第244-267行 `build()` 方法

**原因**：
```typescript
Column() {
  Blank()  // ← 撑满整个屏幕
  Button('清除轨迹')
}
.width('100%')  // ← Column 宽度100%
.height('100%') // ← Column 高度100%
```

`Column` 设置了 `width('100%')` 和 `height('100%')`，加上里面的 `Blank()`，导致整个屏幕被一个**透明的 Column 覆盖**，所有地图的手势事件都被这个透明层拦截了！

**解决方案**：
- 修改 Column 的布局，让它不覆盖整个屏幕
- 让 Column 只覆盖底部按钮区域
- 使用 `hitTestMode` 或调整布局结构

---

## 三、修复方案

### 修复1：移除冲突的定位监听

**修改文件**：`entry/src/main/ets/pages/Index.ets`

**改动内容**：

1. 移除 `startAutoTrack()` 方法中的 `geoLocationManager.on('locationChange')`

2. 改用 Map Kit 自带的定位回调来记录轨迹：
```typescript
// 使用 Map Kit 的位置更新回调
this.mapEventManager?.on('locationChange', (location) => {
  // 记录轨迹点
});
```

或者：保留自己的定位监听，但**不干扰相机移动**：
- 只用于收集轨迹点数据
- 不调用任何相机移动方法

---

### 修复2：修改底部按钮布局

**修改文件**：`entry/src/main/ets/pages/Index.ets`

**改动内容**：

修改 `build()` 方法中的 Column 布局，让它不覆盖整个屏幕：

**方案A**：固定高度，只覆盖底部
```typescript
Column() {
  Button('清除轨迹')
    .backgroundColor('#333333')
    .width(120)
    .height(40)
    .borderRadius(20)
}
.width('100%')
.height(70)  // ← 固定高度，只覆盖底部70像素
.alignItems(HorizontalAlign.Center)
.justifyContent(FlexAlign.End)
.margin({ bottom: 30 })
```

**方案B**：使用 Stack + position 定位
```typescript
Stack() {
  MapComponent(...)
  
  Button('清除轨迹')
    .backgroundColor('#333333')
    .width(120)
    .height(40)
    .borderRadius(20)
    .position({ x: '50%', y: '90%' })  // ← 绝对定位到底部
    .markAnchor({ x: '50%', y: '100%' })  // ← 锚点居中
}
```

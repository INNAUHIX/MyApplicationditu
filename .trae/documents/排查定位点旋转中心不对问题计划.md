# 排查定位点旋转中心不对问题计划

## 一、问题精确定位

**用户描述**:
- ✅ 蓝点和方向箭头存在
- ❌ 旋转时**相机围绕的中心点 ≠ 蓝点位置**
- 用户看到两个点: 一个是蓝点(定位点),另一个是相机围绕的"中心"

---

## 二、根本原因

### 原因: 初始相机位置和定位点位置不一致

**代码分析** ([Index.ets 第35-52行](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L35-L52)):

```typescript
this.mapOptions = {
  position: {
    target: { latitude: 39.9, longitude: 116.4 },  // ← 初始在北京
    zoom: 10
  }
};
```

**问题**:
1. 地图初始化时,相机中心在 (39.9, 116.4) 北京
2. 启用 `setMyLocationEnabled(true)` + `FOLLOW_ROTATE` 后:
   - 蓝点会显示在用户实际位置(例如上海)
   - **但相机中心可能没有立即跟随移动到定位点**
   - 表现: 蓝点在屏幕上的某个位置,不在中心
3. 当用户旋转手机时:
   - 相机围绕**屏幕中心**旋转
   - 屏幕中心 ≠ 蓝点位置
   - 用户看到两个"定位点"不一致

### 原因 2: 我自己加的 cameraChange 监听

虽然 `cameraChange` 监听本身不会移动相机,但它**会触发轨迹记录逻辑**:
- `recordTrackPoint` 会更新 polyline
- 频繁更新可能间接影响相机的平滑移动

**但这可能是次要原因**,主要原因是**相机初始位置问题**。

---

## 三、修复方案

### 核心思路

1. **启动时,先用 Location Kit 获取真实位置**
2. **用 moveCamera 把相机移动到真实位置**(初始就以用户位置为中心)
3. **然后启用 Map Kit 的内置定位**(只负责显示)
4. **完全不监听 cameraChange**(避免干扰)

### 修改1: 删除 cameraChange 监听

**删除** [Index.ets 第157-160行](file:///d:/MyApplicationditu/entry/src/main/ets/pages/Index.ets#L157-L160)

### 修改2: 启动时移动相机到真实位置

在 `enableMyLocation()` 中,先获取真实位置,然后移动相机:

```typescript
private async enableMyLocation(): Promise<void> {
  if (!this.mapController) {
    return;
  }

  try {
    // 1. 先获取真实位置
    let location: geoLocationManager.Location = await geoLocationManager.getCurrentLocation();
    
    // 2. 转换坐标 WGS84 → GCJ02
    let latLng: mapCommon.LatLng = await map.convertCoordinate(
      mapCommon.CoordinateType.WGS84,
      mapCommon.CoordinateType.GCJ02,
      location
    );

    // 3. 把相机移动到真实位置(初始就让中心 = 用户位置)
    let cameraPosition: mapCommon.CameraPosition = {
      target: latLng,
      zoom: 15
    };
    this.mapController?.moveCamera(map.newCameraPosition(cameraPosition));

    // 4. 设置定位点样式
    let style: mapCommon.MyLocationStyle = {
      anchorU: 0.5,
      anchorV: 0.5,
      radiusFillColor: 0x00000000,
      displayType: mapCommon.MyLocationDisplayType.FOLLOW_ROTATE
    };
    await this.mapController.setMyLocationStyle(style);

    // 5. 启用内置定位(只负责显示蓝点和方向箭头)
    this.mapController.setMyLocationEnabled(true);
    this.mapController.setMyLocationControlsEnabled(true);

    // 6. 启动轨迹记录(Location Kit 监听,不调用相机方法)
    this.startAutoTrack();
  } catch (err) {
    console.error(`enableMyLocation failed: ${err}`);
  }
}
```

### 修改3: 添加 startAutoTrack 方法

```typescript
private startAutoTrack(): void {
  try {
    let requestInfo: geoLocationManager.LocationRequest = {
      priority: geoLocationManager.LocationRequestPriority.ACCURACY,
      scenario: geoLocationManager.LocationRequestScenario.NAVIGATION,
      timeInterval: 2,
      distanceInterval: 5
    };

    geoLocationManager.on('locationChange', requestInfo, (location: geoLocationManager.Location) => {
      map.convertCoordinate(
        mapCommon.CoordinateType.WGS84,
        mapCommon.CoordinateType.GCJ02,
        location
      ).then((latLng: mapCommon.LatLng) => {
        this.recordTrackPoint(latLng);
      });
    });
  } catch (err) {
    console.error(`startAutoTrack failed: ${err}`);
  }
}
```

## 四、关键设计

### 为什么启动时要 moveCamera?

- **没有这一步**: 相机中心在北京(39.9, 116.4),蓝点在用户位置
- **有这一步**: 相机中心直接跳到用户位置,蓝点也在用户位置,**两者重合**
- 后续 `FOLLOW_ROTATE` 模式下,相机围绕定位点旋转 = 围绕屏幕中心旋转 ✅

### 为什么不一直用 moveCamera?

- 启动时调用一次,让相机初始位置正确
- 之后用 Map Kit 的 `FOLLOW_ROTATE` 自动跟随
- **不调用重复的 moveCamera**,避免相机跳动

### 为什么 Location Kit 不会冲突?

- 启动时 `getCurrentLocation` 调用一次,用于初始化相机
- 之后 `on('locationChange')` 用于轨迹记录
- **两次都是只读,不调用 moveCamera** (除了初始化那一次)
- Map Kit 内置定位(`setMyLocationEnabled`)独立工作,处理相机跟随

## 六、决策说明

### 为什么不用纯 Map Kit 内置定位?

- 如果**完全依赖** `setMyLocationEnabled`,Map Kit 会**异步**移动相机到定位点
- 在移动完成前,旋转中心 ≠ 蓝点
- 用户看到的就是"两个定位点"
- **手动 moveCamera 一次**,可以**同步**把相机中心设到用户位置

### 为什么 moveCamera 不会和 Map Kit 冲突?

- 我们在 `setMyLocationEnabled(true)` **之前**调用 moveCamera
- 之后由 `FOLLOW_ROTATE` 自动控制相机
- **只调用一次**,不重复

### 为什么要删除 cameraChange 监听?

- 用户明确怀疑"自己加的代码干扰"
- 完全移除自己的代码,**只调用官方 SDK 提供的标准方法**
- 轨迹记录改用 Location Kit,职责更清晰

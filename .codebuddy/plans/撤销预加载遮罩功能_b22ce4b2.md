## 用户需求

撤销当前 Index 页面的「预加载遮罩」功能：进入应用后不再显示 loading 遮罩，地图直接呈现。

## 产品概述

用户此前选择自定义地图样式（暗夜/浅白），但冷启动时会出现「先显示官方默认地图、再切换到自定义样式」的过渡。团队为此加了顶层遮罩预加载机制，但该机制无法根治（根因为云端 styleId 存在网络延迟，遮罩揭开时机早于云端样式上屏）。用户确认不采用本地 styleContent 方案，并要求直接撤掉预加载遮罩。

## 核心功能

- 移除应用冷启动时的全屏 loading 遮罩与底层状态机（mapRevealed / mapLoaded / styleReady / 两个定时器 / 相关方法）。
- 进入应用后地图组件直接显示，无预加载过渡界面。
- 保留：云端 styleId 自定义样式（暗夜/浅白/经典）的切换逻辑、前后台 show()/hide() 生命周期、地图其余功能均不受影响。
- 已知行为（不在本次范围）：因样式为云端 styleId，冷启动仍会直接看到「官方默认 → 自定义」的短暂过渡，用户已知并接受。

## 技术栈

HarmonyOS NEXT（API 12+）/ ArkTS / ArkUI，地图能力基于 Map Kit（@kit.MapKit）。

## 实现策略

采用「纯删除」策略，移除预加载遮罩机制的全部相关代码，不引入新逻辑、不改地图样式套用链路：

- 删除预加载相关的状态声明、兜底定时器、mapLoad 监听、markStyleReady/tryReveal/forceReveal 三个方法及 build 中的遮罩 UI 块。
- applyMapStyleToMap 仅移除内部的 `this.markStyleReady()` 调用，保留 setCustomMapStyle 三分支（暗夜/浅白/经典）的真实套用逻辑。
- 撤销而非修复：根因是云端 styleId 的网络延迟，遮罩无法根治；用户已决定不导出本地 styleContent JSON，故直接撤掉该功能。

## 关键决策与取舍

- 不触碰 initMapOptions 的 styleId + dayNightMode：这是官方支持的初始化即带样式字段，删除会导致重进后样式丢失。
- 不触碰 switchMapStyle / onPageShow / onPageHide：前者负责样式切换持久化，后者是前后台资源生命周期，均与预加载无关。
- MapComponent 的 `.opacity(1)` 保留（无害，且避免回归到早期 opacity(0) 导致地图不加载的问题）。

## 实现注意

- 删除字段/方法后，必须同步清理所有引用点，否则 ArkTSCheck 会报「属性不存在」。已逐一核对引用位置（见下方目录结构与代码位置）。
- aboutToDisappear 中清理定时器的代码块需一并删除，但定位监听清理（geoLocationManager.off）必须保留。
- 删除 markStyleReady 调用后，applyMapStyleToMap 的 .then/.catch 可能变为空块，属合法语法，不影响样式套用。

## 架构设计

本次为「功能回退」型修改，不涉及架构调整。地图渲染、样式套用、生命周期三条链路保持原样，仅移除附带的遮罩状态机。无新增组件或模块。

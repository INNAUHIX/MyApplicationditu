## 产品概述

对当前全自动足迹记录App的核心记录逻辑进行专项优化。聚焦于三档定位模式切换、批量写入、精度过滤和增量覆盖物更新（折线 + 足迹点海量点）。不改展示方案（继续使用华为官方 MapPolyline + MassPointOverlay），不增加统计/城市识别/壁纸等增强功能。

**与初版方案的关键修正**：

- 移除「夜间自动降频」（用户要求后续单独讨论，本次不做）。
- 记录逻辑采用 **Option B**：折线只受「模式时间隔 + 精度过滤 + 5m 位移地板」控制；「记录距离门槛」只控制足迹点（海量点）的稀疏度，不影响折线。
- 模式参数对齐用户最终确认值：省电 200m/30s/≤100m、平衡 50m/10s/≤80m（默认）、耗电 20m/3s/≤50m。

## 核心功能

1. **三档定位模式（按功耗分级，Option B）**：用户在"我的"页面切换；折线按模式时间隔采样（加精度过滤+5m地板），足迹点按模式距离门槛稀疏记录

- 省电：距离 200m / 时间 30s / 精度 ≤100m
- 平衡（默认）：距离 50m / 时间 10s / 精度 ≤80m
- 耗电：距离 20m / 时间 3s / 精度 ≤50m

2. **静止检测降频（沿用现有逻辑）**：连续10个点在15m半径内判定静止，定位间隔降频到30s；恢复移动时回到当前模式间隔
3. **批量写入DB**：缓冲区内积累15个点或每8秒一次批量 INSERT（batchInsert），替代逐行写入
4. **精度过滤**：定位回调中 accuracy 超出当前模式阈值则整点丢弃（折线/足迹点都不记），修复静止时针状漂移
5. **增量覆盖物更新**：折线(setPoints)与足迹点(buildMassPointItems+setItems)均改为节流刷新（每30秒或每50/20个点一次）
6. **数据模型升级**：DB增加 timestamp/accuracy/speed 字段，通过 ALTER TABLE 兼容旧数据
7. **UI调整**：Mine.ets 中"我的卡项"替换为"定位模式"选择器，点击弹出三档选项（省电/平衡/耗电）

## 技术方案

### 一、数据模型升级（Index.ets）

```sql
CREATE TABLE IF NOT EXISTS track_points (
  id INTEGER PRIMARY KEY AUTOINCREMENT, latitude REAL, longitude REAL,
  timestamp INTEGER DEFAULT 0, accuracy REAL DEFAULT 0, speed REAL DEFAULT 0
);
-- 旧库升级（initDatabase 中 try，忽略重复列错误）
ALTER TABLE track_points ADD COLUMN timestamp INTEGER DEFAULT 0;
ALTER TABLE track_points ADD COLUMN accuracy REAL DEFAULT 0;
ALTER TABLE track_points ADD COLUMN speed REAL DEFAULT 0;
CREATE INDEX IF NOT EXISTS idx_ts ON track_points(timestamp);
```

### 二、三档定位模式（Index.ets，Option B）

```typescript
enum RecordMode { POWER_SAVING=0, BALANCED=1, HIGH_POWER=2 }
interface RecordModeConfig { timeInterval:number; distanceInterval:number; maxAccuracy:number; label:string; }
const MODE_CONFIG: RecordModeConfig[] = [
  { timeInterval:30, distanceInterval:200, maxAccuracy:100, label:'省电' },
  { timeInterval:10, distanceInterval:50,  maxAccuracy:80,  label:'平衡' },
  { timeInterval:3,  distanceInterval:20,  maxAccuracy:50,  label:'耗电' }
];
@State currentMode: RecordMode = RecordMode.BALANCED;

// 折线位移地板：与上一个折线点距离 < 5m 视为原地抖动，丢弃
private static readonly MIN_TRACK_DISTANCE: number = 5;
```

- `getModeInterval()` → 返回 `MODE_CONFIG[currentMode].timeInterval`
- `switchMode(mode)` → 设置 currentMode 并 `switchLocationInterval(getModeInterval())`（静止时保持静止间隔）
- 定位回调中：精度过滤（accuracy>maxAccuracy 丢弃）→ 位移地板（<5m 丢弃）→ 记录

### 三、记录逻辑（Option B）

```
recordTrackPoint(gcj02, accuracy, speed):
  1. 折线：每通过「精度过滤 + 5m地板」的点都 push 进 trackPoints（保持平滑）
  2. 足迹点：距上一个足迹点直线距离 ≥ MODE_CONFIG[mode].distanceInterval 才 push 进 footprintPoints
  3. 入 pendingPoints 缓冲；达 BATCH_SIZE(15) 立即 flushBatch()
  4. refreshOverlayOptimized() 节流刷新覆盖物
```

### 四、批量写入（Index.ets）

```typescript
private pendingPoints: TrackPointRecord[] = [];
private flushTimer: number | null = null;
private static readonly BATCH_SIZE = 15;
private static readonly FLUSH_INTERVAL = 8000;

private flushBatch(): void {
  if (!this.rdbStore || this.pendingPoints.length === 0) return;
  let values = this.pendingPoints.map(p => ({latitude,longitude,timestamp,accuracy,speed}));
  this.rdbStore.batchInsert(TABLE_NAME, values).then(...).catch(...);
  this.pendingPoints = [];
}
```

aboutToAppear 启动 `setInterval(() => this.flushBatch(), FLUSH_INTERVAL)`；aboutToDisappear 清除。

### 五、增量覆盖物更新（Index.ets）

```typescript
private lastPolylineUpdatePointCount = 0;
private lastFootprintUpdateCount = 0;
private lastOverlayUpdateTs = 0;
private static readonly OVERLAY_UPDATE_INTERVAL = 30;
private static readonly POLYLINE_UPDATE_POINT_GAP = 50;

```

`refreshOverlayOptimized()`：按 displayMode 分别判断 gap 与时间间隔，决定是否调用 updatePolyline / updateMassPointOverlay；两个 update 方法内同步更新对应计数。

（夜间自动降频已移除，不在本次范围。）

### 六、Mine.ets UI改造

"我的卡项"CardItem → "定位模式"CardItem（图标 timer），右侧 statusText 显示当前模式；点击弹出三档选择（省电/平衡/耗电，复用 displayModePopupBuilder 的 bindPopup 模式，选中项打勾）；选择后通过 `onRecordModeChange(modeIndex)` 回调 Index.ets 的 `switchMode()`。

### 七、不影响现有功能

地图工具面板、设置面板、沉浸模式、日夜切换、地图样式切换、清除轨迹、海量点/折线切换均不变。

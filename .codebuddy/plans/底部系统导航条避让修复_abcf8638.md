## 用户需求

解决"内容不避让系统底部导航条"的问题。现象与之前状态栏同类：应用未对屏幕最底部的系统导航条（手势/返回主页条）做避让，导致沉浸全屏的地图内容、悬浮底栏与底部导航条相互重叠，露出实色占位底条。

## 产品概述

应用当前已对顶部状态栏做了标准避让（透明状态栏 + 顶部 UI 按 `statusBarHeight` 偏移）。本次沿用同样范式，把底部系统导航条纳入避让：使其透明覆盖在地图之上，并让悬浮底栏与地图内置控件整体上抬一个导航条高度，避免重叠。

## 核心功能

- 系统导航条透明沉浸：消除底部实色占位条，地图画面延伸至导航条下方。
- 导航条高度读取：通过窗口 `avoidArea.bottomRect.height` 取得底部安全区高度。
- 悬浮底栏避让：HdsTabs 浮条 `barBottomMargin` 叠加导航条高度，整体抬过导航条。
- 地图控件避让：地图 `setPadding` 底部占位叠加导航条高度（含沉浸模式切换），缩放/我的位置按钮抬到浮条与导航条之上。

## 技术栈

- 平台：HarmonyOS NEXT（ArkTS + ArkUI），窗口能力 `@kit.ArkUI`（`window.SystemBarProperties` / `getWindowAvoidArea`），UI 组件 `@kit.UIDesignKit`（`HdsTabs` 悬浮样式）。
- 复用现有范式：完全对齐已落地的状态栏处理（透明状态栏 + 按 `statusBarHeight` 偏移 UI），仅把"顶部"对应为"底部"，不引入新架构、不新增文件。

## 实现方案

### 1. 系统导航条透明化（EntryAbility.ets）

在 `onWindowStageCreate` 的 `setWindowSystemBarProperties` 中，除已有 `statusBarColor` / `statusBarContentColor` 外，补充导航条属性，使其透明覆盖于地图之上（与状态栏透明同理），消除实色占位条：

- `navigationBarColor: '#00000000'`
- `navigationBarContentColor: '#FFFFFF'`（地图画面偏暗，手势/指示图标用浅色保证可见；若后续地图浅色场景多，可按深浅模式动态取色）

### 2. 读取底部安全区高度（Index.ets）

- 新增 `@State navBarHeight: number = 0;`（与 `statusBarHeight` 并列，第 133 行附近）。
- 在第 323-324 行的 `getWindowAvoidArea` 回调内，补充读取 `avoidArea.bottomRect.height`：
`this.navBarHeight = px2vp(avoidArea.bottomRect.height);`
- 该回调为异步，地图可能在 `navBarHeight` 就绪前已 `onInit`。为保险，在同一回调内若 `this.mapController` 已存在，立即重新调用一次 `setPadding` 使底部占位生效（见第 4 点方法）。

### 3. 悬浮底栏避让（Index.ets，第 1731-1740 行）

`HdsTabs` 的 `.barFloatingStyle({ barBottomMargin: 28, ... })` 中，将 `barBottomMargin: 28` 改为 `barBottomMargin: 28 + this.navBarHeight`，使浮条整体抬过系统导航条，避免两者重叠/手势冲突。

### 4. 地图控件避让（Index.ets，第 777-782 行与第 827-832 行）

地图内置控件（缩放、"我的位置"）靠 `setPadding({ bottom })` 上抬，原硬编码 `90` 未含导航条高度。将两处 `bottom` 改为 `this.navBarHeight + 90`：

- 初始 `setPadding`：`bottom: this.navBarHeight + 90`
- 沉浸切换 `mapClick` 回调：`bottom: next ? 0 : this.navBarHeight + 90`
- 配合第 2 点的异步重应用，保证 `navBarHeight` 异步到达后占位正确。

## 实现注意

- 性能/可靠性：`getWindowAvoidArea` 仅增加一次轻量读取，无热路径开销；`setPadding` 重应用仅一次，不影响地图渲染帧率。
- 向后兼容：默认 `navBarHeight=0`，若该回调异常未赋值，退化为原 `90` 行为，不破坏现有功能；`try/catch` 已覆盖状态栏，导航条新增字段同样在既有 `setWindowSystemBarProperties` 的 `try` 内，异常仅日志不阻塞。
- 范围控制：仅改动 EntryAbility.ets 与 Index.ets 两处，不影响轨迹记录、时间视图、地图样式等既有逻辑；不改 `ignoreLayoutSafeArea([SYSTEM],[TOP,BOTTOM])`（沉浸全屏延伸仍需要）。
- 度量单位：`avoidArea.bottomRect.height` 为 px，统一用 `px2vp` 转 vp，与状态栏 `statusBarHeight` 处理一致；`setPadding` 入参为 vp，直接相加即可。

## 架构设计

沿用现有"透明系统栏 + 安全区高度状态 + UI 偏移"的单层避让模型，仅扩展到底部：

```mermaid
flowchart TD
    A[EntryAbility 设导航条透明] --> B[系统导航条透明覆盖地图]
    C[Index avoidArea 读 bottomRect.height] --> D[@State navBarHeight]
    D --> E[HdsTabs barBottomMargin = 28 + navBarHeight]
    D --> F[地图 setPadding bottom = navBarHeight + 90]
    E --> G[浮条抬过导航条]
    F --> H[地图控件避让导航条]
```

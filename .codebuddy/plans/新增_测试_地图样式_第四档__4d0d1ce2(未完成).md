## 用户需求

将用户提供的最新版浅色极简地图样式 JSON（共 90 条规则，含隐藏道路编号图标、保留路名文字与购物中心色块等设定）接入应用，成为继「经典 / 暗夜 / 浅白」之后的**第四个地图样式**，样式名称命名为「测试」。

## 产品概述

在「地图工具」面板的「地图样式」分段控件中新增第四个选项「测试」。选择后地图底图切换为这套浅色极简自定义样式（固定白天模式），保留道路与路名、水系、绿地、建筑、铁路/轮渡/机场线，隐藏 POI、行政文字、公交文字、道路编号图标等。

## 核心功能

- 新增样式常量：以 ArkTS 对象数组形式保存该 90 条样式规则（MAP_STYLE_TEST），避免手写转义字符串。
- 「地图样式」分段按钮新增「测试」选项，并将控件宽度由 160 调宽至约 200 以容纳四档。
- 初始化阶段：索引 3 视为浅色样式，dayNightMode 使用 DAY，styleId 保持 undefined（沿用现有经典档行为，地图就绪后由 applyMapStyleToMap 用 styleContent 应用）。
- applyMapStyleToMap 新增 index===3 分支：固定 DAY 模式 + setCustomMapStyle({ styleContent: JSON.stringify(MAP_STYLE_TEST) })，并补充日志。
- 持久化逻辑（存 index）无需改动，默认样式仍为 0=经典，不影响现有用户。

## 技术栈

- 平台：HarmonyOS NEXT（ArkTS + ArkUI），地图使用 @kit.MapKit。
- 样式注入方式：`map.MapComponentController.setCustomMapStyle({ styleContent })` 接收 JSON 字符串；现有「经典」档已用 `styleContent: '[]'` 验证可行，新增档沿用同一机制，无需在 Petal Maps Studio 发布样式 ID。

## 实现方案

1. **新增样式常量**：在 Index.ets 顶部（MAP_STYLE_LIGHT_ID 之后）定义 `const MAP_STYLE_TEST: object[] = [ ... ]`，将用户提供的 90 条规则原样录入为对象数组。使用 `JSON.stringify()` 转字符串注入，规避手动转义 `\u0026quot;` 等错误。
2. **切换控件扩展**：`mapStyleSegmentOptions` 的 `buttons` 增加第 4 项 `{ text: '测试' }`；其 UI 处 `.width(160)` 改为 `.width(200)`，确保四档短标签完整显示且不挤压。
3. **初始化参数**：`initMapOptions()` 中 `initialDayNight` 由 `index === 2 ? DAY : NIGHT` 改为 `index === 2 || index === 3 ? DAY : NIGHT`；`initialStyleId` 维持 `index===1→暗夜ID, index===2→浅白ID, 否则 undefined`，索引 3 自然落入 undefined 分支（mapOptions 不支持 styleContent 初始化，与经典档首帧行为一致，可接受）。
4. **核心应用分支**：`applyMapStyleToMap(index)` 现有 `if(1)/else if(2)/else(经典)` 之后新增 `else if (index === 3)`：先 `controller.setDayNightMode(mapCommon.DayNightMode.DAY)`，再 `controller.setCustomMapStyle({ styleContent: JSON.stringify(MAP_STYLE_TEST) })`，并加 `console.info` 日志与圆括号内 catch（与暗夜/浅白分支格式一致）。
5. **持久化兼容**：`saveMapStyle/loadMapStyle` 仅存读 index，新增 3 完全兼容，无需改动；默认 `mapStyleIndex = 0` 不变。

## 实现注意

- 单文件改动集中（仅 Index.ets），不改变现有三档样式任何行为， blast radius 可控。
- JSON 数组为静态常量，仅在切换/初始化时 `JSON.stringify` 一次，无热路径性能问题；可进一步缓存字符串，但当前规模无需。
- 错误兜底：setCustomMapStyle 失败 catch 后仅打日志，不阻塞其他样式分支；若 styleContent 解析失败，地图会回退到内置样式，不影响定位与轨迹记录。

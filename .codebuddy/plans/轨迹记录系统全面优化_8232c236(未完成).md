## 产品概述

将当前简单的轨迹记录 App 升级为类似"一生足迹"的完整足迹管理 App，实现轨迹记录的全面控制、智能分段展示、丰富的数据统计和长线性能优化。

## 核心功能

### 1. 记录控制（开始/暂停/继续）

- 地图上一键控制记录状态：一键开始/暂停/继续
- 三种状态：空闲(灰色) → 记录中(红色脉冲动画) → 暂停(黄色)
- 暂停时降频定位但不关闭，恢复时自动切回高频

### 2. 数据模型增强

- 数据库增加时间戳、精度、速度、会话ID字段
- 新增记录会话表，记录每次会话的起止时间、里程、时长
- 兼容升级：自动检测旧表结构并迁移历史数据

### 3. 轨迹分段展示

- 按天分段：当天红色、前一天橙色、更早渐变灰色
- 支持日期选择：左右箭头切换查看特定日期的轨迹
- 折线样式优化：半透明底色+实线描边、测地线模式

### 4. 统计信息展示

- 主页面显示今日里程和记录时长小卡片
- "我的"页面顶部四宫格统计卡片：总里程、记录天数、今日里程、累计点数
- 全部基于数据库实时聚合，不依赖内存数据

### 5. 性能优化

- 懒加载：只加载最近30天数据，按日期范围分批查询
- 点抽稀：低缩放级别时简化折线点数
- 批量写入：5个点或3秒一批次落库，减少DB写入压力

## 技术栈

- 框架：HarmonyOS NEXT (API 12+) / ArkTS
- 地图：@kit.MapKit
- 定位：@kit.LocationKit
- 数据库：@kit.ArkData (relationalStore)
- 后台长任务：@kit.BackgroundTasksKit
- UI：ArkUI 声明式组件 + @kit.UIDesignKit

## 总体架构

将轨迹逻辑从 Index.ets 拆分为独立 Service 层，保持页面组件聚焦于 UI 编排。

### 模块划分

```
entry/src/main/ets/
├── model/
│   ├── TrackData.ets              # [NEW] 类型定义
│   └── RecordState.ets            # [NEW] 状态机枚举+控制接口
├── services/
│   ├── TrackDatabase.ets          # [NEW] 数据库层
│   ├── TrackRecorder.ets          # [NEW] 记录状态机 + 定位管理
│   └── StatisticsCalculator.ets   # [NEW] 统计计算
└── pages/
    ├── Index.ets                  # [MODIFY] 大幅精简，引用 Service
    └── Mine.ets                   # [MODIFY] 增加统计卡片
```

### 关键设计决策

1. **状态机分离**：TrackRecorder 独立管理 idle/recording/paused 状态、定位监听的注册注销和频率切换，Index 只通过回调接收新点。

2. **数据迁移**：检测旧表 `track_points` 是否缺少 timestamp/session_id 列，缺少则 ALTER TABLE ADD COLUMN，并将历史记录的 timestamp 基于 id 线性插值估算，session_id 设为 'legacy'。

3. **批量写入**：TrackDatabase 内部维护 pending 队列，收集 5 个点或每 3 秒 flush 一次，减少 DB 写入次数。

4. **按天分色折线**：维护 `Map<string, MapPolyline>` 以日期为 key 管理多条折线，当天红色、前一天橙色、更早灰色渐变。切换日期时显示/隐藏对应折线。

5. **统计缓存**：在 TrackDatabase 中维护每日里程缓存表，插入点时增量更新，避免每次统计全表扫描。

### 数据库增强设计

```sql
-- 轨迹点表（增强字段）
CREATE TABLE IF NOT EXISTS track_points (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  latitude REAL NOT NULL,
  longitude REAL NOT NULL,
  timestamp INTEGER NOT NULL DEFAULT 0,    -- Unix timestamp in ms
  accuracy REAL DEFAULT 0,                  -- GPS accuracy in meters
  speed REAL DEFAULT 0,                     -- speed in m/s
  session_id TEXT DEFAULT 'legacy'          -- recording session UUID
);
CREATE INDEX IF NOT EXISTS idx_timestamp ON track_points(timestamp);
CREATE INDEX IF NOT EXISTS idx_session ON track_points(session_id);

-- 记录会话表（新增）
CREATE TABLE IF NOT EXISTS track_sessions (
  id TEXT PRIMARY KEY,                      -- session UUID
  start_time INTEGER NOT NULL,
  end_time INTEGER DEFAULT 0,
  distance REAL DEFAULT 0,
  duration INTEGER DEFAULT 0,
  point_count INTEGER DEFAULT 0
);
```

### 性能考量

- **加载优化**：启动时只加载最近30天的点（`WHERE timestamp > ?` + `ORDER BY timestamp`），按天分组成多个 MapPolyline
- **抽稀策略**：缩放级别 < 12 时，对当天折线每10个点取1个做简化显示；缩放到14以上显示全部
- **内存控制**：按天存储点列表，单天点数上限约17万点（1秒一个点），约70MB内存；实际可通过5m距离过滤减少到1/5
- **写入优化**：批量写入每次5条或每3秒flush，减少DB事务次数约20倍

### 实现注意事项

- 数据迁移不影响现有用户数据
- Index.ets 只替换轨迹相关逻辑，不破坏现有地图工具面板/设置面板/我的面板等 UI 结构
- Service 层使用独立 TAG 日志，方便排查

## 设计风格

简约现代风格，以地图为核心，叠加轻量级半透明控件。记录按钮使用脉冲动画表示活跃状态，日期切换器采用磨砂玻璃背景的悬浮胶囊。统计卡片使用毛玻璃质感、圆角大卡片设计。

### 页面结构

#### 1. 足迹 Tab（主地图页）

- **记录控制按钮**：右下角悬浮按钮，直径56px圆角
- 记录中：红色背景，白色暂停图标 + 脉动光圈动画
- 暂停：黄色背景，白色播放图标
- 空闲：灰色背景，白色录制图标
- **日期选择器**：左上角半透明磨砂玻璃胶囊，显示"今天 / 7月11日" + 左右箭头按钮
- **今日统计小卡片**：日期选择器下方，紧凑显示"今日 3.2 km · 记录 2h15m"
- **地图**：全屏，叠加按天分色的轨迹折线

#### 2. 我的 Tab（半模态面板）

- **统计封面卡片**：顶部毛玻璃质感卡片，四宫格布局
- 总里程 / 记录天数 / 今日里程 / 累计点数
- 每个格子上方是大号数字，下方是小字标签
- 下方保持现有列表项（定位权限、设置等）

### 新旧功能对比

- 新增：记录控制按钮（右下角）、日期选择器（左上角）、今日统计卡片
- 保留：地图工具面板（轨迹切换/地图样式）、设置面板、定位权限面板、沉浸模式
